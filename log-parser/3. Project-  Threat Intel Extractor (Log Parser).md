

**Description:** A security automation tool that parses raw Linux server logs to detect brute-force attacks and extract the IP addresses of the threat actors.

**Core Logic:**

- Safely opens a specified log file (e.g., `auth.log`) and iterates through it line-by-line.
    
- Uses string matching (`in` keyword) to check for specific failure indicators (e.g., "Failed password").
    
- Employs `.split()` to convert log lines into lists, and `.index()` to locate an anchor word (like "from").
    
- Dynamically extracts the attacker's IP and username relative to the anchor word.
    
- Stores extracted data in a Python `set()` to automatically eliminate duplicate entries, then appends the unique threats to a `blacklist.txt` file.
    

**Engineering Learnings:**

- **File Iteration:** Always iterate over the opened file object (`for line in f:`), not the string name of the file.
    
- **Strict Case Sensitivity:** Python string matching is ruthless; searching for "failed" will not match "Failed".
    
- **Anchor Extraction:** Instead of blindly counting word positions, finding a static anchor word ensures accurate data extraction even if the log formatting shifts slightly.
