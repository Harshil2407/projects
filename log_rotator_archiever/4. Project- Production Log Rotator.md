**Description:** An infrastructure resilience script that safely archives, compresses, and purges massive application logs without interrupting the active web server processes writing to them.

**Core Logic:**

- Initializes the `logging` module to maintain a permanent, timestamped audit trail of all script actions.
    
- **Rotation Phase:** Uses `shutil.copy2()` to duplicate the active `app.log` file, dynamically naming the copy with the current timestamp.
    
- **Truncation Phase:** Immediately opens the original active log in write (`"w"`) mode and uses `.truncate(0)` to empty it, allowing the server to continue writing without losing its file descriptor.
    
- **Compression Phase:** Converts the copied log into a highly compressed archive using `gzip.open()`.
    
- **Purge Phase:** Uses `.glob("*.gz")` to scan the directory for archives, calculates their exact age using the `datetime` module, and uses `os.remove()` to permanently delete files older than the 7-day retention policy.
    
- Wraps volatile file operations in `try-except` blocks to prevent the script from crashing due to unexpected OS permissions.
    

**Engineering Learnings:**

- **Bypassing File Locks:** Active applications lock their files. Moving an active log causes errors; copying and truncating is the DevSecOps standard for live state management.
    
- **Time Math:** The `datetime` module allows scripts to convert system timestamps (`st_mtime`) into human-readable math to enforce data retention policies.
    
- **Professional Audit Trails:** `print()` statements vanish when a terminal closes. The `logging` module is mandatory for unattended automation tasks (like cron jobs) to ensure administrators can trace failures.
    
- **Resilient Automation:** Using specific `try-except` blocks (like `PermissionError`) stops a script from completely dying when it hits a single bad file.
