

**Description:** An automated File Integrity Monitoring (FIM) script designed to detect unauthorized file modifications within a secure directory.

**Core Logic:**

- Employs the `argparse` library to handle multiple complex terminal flags (`-d` for directory, `--baseline` for snapshot generation, `--audit` for security checking).
    
- **Baseline Mode:** Scans the target folder, extracts file sizes, and saves them to a `baseline.txt` file in a structured `key|value` format (Filename | Size).
    
- **Audit Mode:** Loads the `baseline.txt` data into a Python dictionary in memory. It then rescans the live folder and compares current file sizes against the expected sizes in the dictionary.
    
- Triggers terminal alerts if a file size changes or if an untracked file is discovered.
    

**Engineering Learnings:**

- **Advanced CLI Routing:** `argparse` is mandatory when a script requires multiple inputs, default values, or mutually exclusive operating modes.
    
- **I/O Performance:** `with open()` blocks should always be placed _outside_ of loops to prevent massive CPU and disk bottlenecks.
    
- **State Management:** Never use `os.chdir()` to change the global directory state of a script. Always use absolute or relative targeting (e.g., `folder.iterdir()`) to prevent catastrophic failures if the script crashes midway.
